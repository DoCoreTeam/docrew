# Requirements — PHASE 0.1 TRADEOFF CHECK (v2.0.31)

## 기능 요구사항
- CEO는 MEDIUM+ 요청 수신 시 SIZE ASSESSMENT 직후 트레이드오프 스캔 실행
- 6개 체크리스트 항목 중 1개 이상 해당 시 `[TRADEOFF DETECTED]` 출력 + 선택지 제시 후 대기
- 해당 없을 시 `[TRADEOFF CLEAR]` 출력 → 자동으로 Q&A 진행
- SMALL 경로는 이 단계 생략 (FAST PATH 직행)

## 체크리스트 (C안 — 1차 자동)
1. 기존 코드 대규모 수정 (500줄+ 영향) — 회귀 리스크
2. 단기 해결책이 장기 아키텍처에 충돌
3. 새 의존성 추가 → 보안/유지보수 비용 증가
4. 요청 방식이 더 많은 버그/복잡도 생성 가능
5. 기존 동작의 브레이킹 체인지 내포
6. 환경 제약으로 실제로 작동 불가한 방식 요청

## 비기능 요구사항
- ceo.md ≤300줄 유지
- 추가 에이전트 호출 없음 (CEO 자체 판단)
- 출력 형식 표준화 (매번 동일 구조)

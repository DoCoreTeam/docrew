# Requirements — v2.0.40 exec-rules

## 기능 요구사항

### F1. 실행 완수 강제
- 구현 시작 전 범위 전체 확정 필수
- 중간 멈춤 / "다음 스프린트로" 제안 금지
- 범위 분리가 필요하면 구현 시작 전(Q&A 단계)에만 사용자 승인 가능

### F2. 완료 검증 강제
- GATE 2에서 `04-completion-criteria.md` 전 항목을 라인별 ✅/❌ 출력
- 미완료 항목 존재 시 → 완료 선언 불가, 에이전트 재작업 지시
- "대충 됐다" 판정 금지 — 문서 근거 없는 완료 선언 = 규칙 위반

### F3. CLI 직접 실행
- 실행 가능한 CLI가 있으면 무조건 직접 사용
- 사용자에게 위임 조건: 사용자 인증(oauth/MFA) 또는 사용자 로컬 전용 환경만
- 그 외 모든 경우 CEO/에이전트가 직접 Bash 실행

### F4. 세션 리포트 필수
- [CEO REPORT] 블록은 GATE 5 이후 절대 생략 불가
- 멀티세션 사용자가 "이번에 뭘 했는지" 한눈에 파악 가능해야 함
- FAST PATH도 [CEO FAST REPORT] 필수 (이미 존재하나 강제도 추가)

## 비기능 요구사항
- 기존 강제 규칙(5개) 오염 금지 — 새 규칙은 별도 섹션으로
- CLAUDE.md 3곳(root/domangcha/~/.claude) 동기화 필수
- error-registry.md에 EXEC-001~004 추가로 GATE 1 자동 탐지
